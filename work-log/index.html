v0.3.0 Jul 10, 2026
NearNosh

Green wordmark restored & swipe-to-clear

  • Reverted the wordmark to its dark-green “Near” / light-green “Nosh” two-tone, unified across the header, splash, and About card on one set of tokens.
  • Filters: swipe the sheet down to clear everything and return to the full list.

The wordmark had drifted to orange — pulled it back to the two greens. Note: because the app is locked to dark mode, the “dark green” renders as a legible deeper green on the dark ground rather than true black-green.

v0.2.0 Jul 10, 2026
NearNosh

Dark mode locked, new icon, amenity tags

  • Locked the app to dark mode — first load and every load, browser and PWA, with no flash of light.
  • New fork-and-knife app icon (replaced the map pin); redrew the settings gear as a clean cog.
  • Added amenity tags — Walk-In Friendly, Cash Only, Outdoor Seating, Quick Meal — backed by a Supabase table you can curate by hand.
  • Added an “Open now” tag (timezone-aware to NYC) and a Breakfast-only filter.

Tags read from Supabase so they’re editable without a deploy; seeded a few (including some that are open now) to preview the look. Caveat: the live Supabase read couldn’t be verified from the sandbox — outbound network is blocked there.

v0.1.0 Jul 9–10, 2026
NearNosh

Initial build-out & first release

  • Applied the “Fresh Market” design system — cream/sage/citrus palette with Fraunces, Space Grotesk, and JetBrains Mono.
  • Flip cards: info-“i” toggle, an “Open in Google Maps” button, list badges, and a blurb on the back.
  • Turned the header into a flippable About card; built two-tier filters plus a Settings menu (units, dietary).
  • Added a Google map view, consolidated the NYT 2023–2026 lists, and wrote a Google Places enrich script.
  • Removed along the way: “Best of the Best,” neighborhood search, the “sorted by distance” label, and the community thumbs-voting UI (backend kept).
  • Shipped as a PWA deploying to Vercel.

Consolidation matches the same spot across lists by name and proximity, then sorts by distance. Coordinates and some list subsets are approximate — flagged honestly in the repo.
